Contributing
============

Contributions are very welcome! This document provides guidelines and instructions for contributing to the project.

Getting Started
---------------

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request

Development Setup
-----------------

See the Installation section for setting up a development environment.

Code Style
----------

The project uses ruff for linting and formatting. Please ensure your code follows the project's style guidelines.

Testing
-------

Please ensure all tests pass before submitting a pull request:

.. code-block:: console

    $ tox

Reporting Issues
----------------

If you encounter any problems, please file an issue with a detailed description.
